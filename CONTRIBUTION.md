# Руководство по оформлению кода

## 1. Структура заголовка файла

Каждый файл должен начинаться с пояснения в следующем формате:

#### { ТИП-ФАЙЛА: Краткий функционал

#### Небольшое описание предназначения }

Типы файлов и их описание:

- VIEW - модули, содержащие классы arcade.View, представляющие экраны игры

- FILE - вспомогательные модули, утилиты, config файлы

- MODULE - пакеты и подмодули

## 2. Уровни комментариев

### Уровень 1: Крупные или логическии выделенные блоки  (# --)

Используется для разделения и описания крупных блоков кода, которые выполняют одну общую задачу, или для выделения функций/классов.

### Уровень 2: Группы связанных строк (# -)

Используется для описания небольших групп связанных строк кода, которые вместе решают конкретную подзадачу.

### Уровень 3: Отдельные строки (#)

Используется для пояснения конкретной строки кода.

## 3. Специальные комментарии (TODO и пометки о доработке)

Используйте стандартные комментарии, которые распознаются PyCharm

### Примеры:

- TODO: Добавить поддержку нескольких слоев тайлов

- FIXME: Исправить утечку памяти при частом зуммировании камеры

- HACK: Временное решение для совместимости со старой версией

## 4. Стили написания кода

**- Все классы пишутся в Верблюжем стиле**

**- Все функции классов в Змеином стиле**

Примечания:

Если функция класса не используется иных файлов, то перед ней следует добавить подчеркивание **(_test_game)**
